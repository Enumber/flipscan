把高拍仪（或任意摄像头）对准书本或试卷，Flipscan 会盯着画面，**检测到你翻页、等画面稳定后自动按下快门**——扫一整本书都不用手动拍。拍到的图会自动处理成清晰的扫描件（白底黑字、保留红章）。

### 特性

- **翻页自动拍照** —— 帧间差分状态机（待机 → 监控中 → 翻页中 → 稳定中 → 拍照中）识别翻页动作，画面稳住后自动快门。
- **扫描增强** —— 逐通道光照归一化（消色偏）+ 背景白化 + 提对比 + 锐化，**红色印章不会被洗掉**。
- **透视裁剪 + 自动转正** —— 找出纸张四角做透视变换拉平，横向照片自动转成竖向。裁剪与扫描增强是两个独立开关；转竖向是裁剪的子选项，只在裁剪打开时才有效。
- **本次/历史相册** —— 缩略图列出刚拍的和以往几次的照片，可点开放大、多选删除、一键删除全部。
- **中英双语界面** —— 跟随系统语言（`LANG` 含 `zh`/`cn` 显示中文，否则英文）。
- **不重启就能换摄像头** —— 在控制面板里挑另一个摄像头，实时画面立刻切过去；勾上「固定这个摄像头」，下次启动就直接用它。
- **整理清楚的设置弹窗** ——「常规」放启动拍摄与摄像头选择，「高级」放翻页检测调优与恢复默认，「其他」放更新设置。关闭后不占拍摄主窗口空间，再打开仍复用同一个弹窗。
- **摄像头接管** —— 被别的程序占用时可以强制夺回、也可以再让出去；良田官方软件会自动暂停并在退出后恢复。
- **可选 AI 分析** —— `analyze_papers.py` 可把页面（图片/PDF/视频）交给 Gemini 分析并打印结果。

### 一键安装（免 sudo）

```bash
bash install.sh
```

有图形界面时会弹出**安装窗口**（无显示器则回退终端问答）：选安装位置（用户目录 / 自定义路径 / 系统级 `/opt`，后者走 `sudo`）、要不要放桌面图标，以及——检测到摄像头时——选一个固定为默认摄像头（写入 `~/.config/flipscan/config.json`，之后随时可在程序内「更改」按钮里换）。一路回车就是默认值。

安装器会自动建虚拟环境（`gemini_env/`）装好 OpenCV，并在**桌面**和**应用程序列表**各放一个可直接双击运行的图标；默认的用户级安装**不需要管理员密码**。非交互运行（管道调用或带任何参数）时行为和以前完全一样——一步到位、不提问。

注意：默认的用户级安装是**原地运行**的，程序就在你克隆下来的这个文件夹里跑——装完别删、别挪这个目录，否则图标会失效。想把文件复制到固定位置请用 `--prefix`。

<details>
<summary>高级选项</summary>

```bash
bash install.sh --system            # 系统级安装（需要管理员）：装到 /opt + /usr/share
bash install.sh --prefix ~/apps     # 把程序复制到 ~/apps/flipscan 再建图标
bash install.sh --no-desktop-icon   # 只进应用菜单，不放桌面图标
bash install.sh --uninstall         # 卸载图标、菜单项与自启动项
bash install.sh --help              # 查看用法
```

`--uninstall`（有显示器时弹出确认窗） 只清理对应模式装的东西，所以卸载系统级安装要写
`bash install.sh --system --uninstall`。`--uninstall` 与 `--prefix`/`--system` 同时给出时，复制过去的程序目录也会一并删除。

### 依赖

Python 3.10+、OpenCV 和 Tk。`install.sh` 会自动把 `opencv-python` 和 `google-generativeai` 装进虚拟环境；Tk 和 v4l-utils 需要用系统包管理器装：

| 发行版 | 需要的系统包 |
|---|---|
| Debian/Ubuntu | `sudo apt install python3-tk v4l-utils` |
| Fedora | `sudo dnf install python3-tkinter v4l-utils` |
| Arch | `sudo pacman -S tk v4l-utils` |

### 使用

桌面图标执行的是 `run.sh --capture`，即打开拍照窗口。命令行用法：

```bash
./run.sh --capture                       # 自动检测/弹窗选择摄像头
./run.sh --capture --camera 1            # 按编号指定
./run.sh --capture --camera /dev/video2  # 或按设备路径指定

./run.sh --images 试卷1.jpg 试卷2.jpg     # AI 分析已有图片或 PDF
./run.sh --video 录像.mp4                # AI 分析视频
```

拍照窗口**只认 `--camera` 这一个参数**，其余全部在界面里操作。不带 `--capture` 时，`run.sh` 会把参数原样交给 `analyze_papers.py`。

**用哪个摄像头**，按这个优先级：

1. 命令行的 `--camera` —— 永远最优先，且**不会**改动你固定下来的设置。
2. 固定的摄像头：勾了「固定这个摄像头」且该设备还在的话，直接用它。
3. 否则走自动检测：名称像高拍仪的优先选中 → 只有一个摄像头就直接用 → 有多个则**弹窗列出所有设备**让你挑。

笔记本内置摄像头通常是 `/dev/video0`，外接高拍仪一般是靠后的编号。

**运行中换摄像头**：控制面板里保存目录的下面一行就是当前摄像头，右边有自己的**更改**按钮。点开会列出所有能找到的采集设备，选一个实时画面立刻切过去——不用重启，已经拍好的照片也不会丢。如果目标设备被占用或已拔掉，程序会保持在原来那个摄像头上，并在预览区提示你。

**默认保存文件夹**：侧栏的**更改**和 **设置 → 拍摄与摄像头 → 默认保存文件夹** 写的是同一条 `output_dir`（`~/.config/flipscan/config.json`），下次启动仍进这个目录。设置窗口会相对**主窗口居中**打开。

**固定摄像头**：勾选控制面板里的**固定这个摄像头**（或启动弹窗里的「记住我的选择」），选择会立即写入：

```
~/.config/flipscan/config.json        # 或 $XDG_CONFIG_HOME/flipscan/config.json
```

```json
{
  "camera": "/dev/video2",
  "remember_camera": true
}
```

取消勾选就回到自动检测。删掉这个文件等于全部重置；文件读不出来或格式坏了，程序会忽略它退回自动检测，**而不是拒绝启动**。

**固定的摄像头不见了**（拔掉了，或者重启后被内核重新编号）——程序不会报错退出，而是在启动时弹出摄像头选择框，并注明原来固定的设备找不到了；你在那里选的设备会成为新的固定项（如果勾选框是开着的）。

**界面上的操作：**

| 控件 | 作用 |
|---|---|
| **▶ 开始记录** | 启动自动拍照，再点一次停止。 |
| **⊙ 拍照** | 立刻手动拍一张，不管有没有在记录。 |
| **更改**（第一行） | 选择保存文件夹（自带带书签的文件夹浏览器，可新建文件夹）。 |
| **更改**（摄像头那行） | 运行中切换到另一个摄像头，实时画面立刻跟着切。 |
| **固定这个摄像头** | 把当前摄像头固定下来，下次启动直接用、不再询问。 |
| **扫描增强（白底黑字·留红章）** | 白底黑字、保留红章，**默认关闭**。 |
| **自动裁剪（裁出纸张）** | 识别纸张四角并拉平。 |
| **转竖向** | 把横向照片转成竖向。是「自动裁剪」的子选项——裁剪关闭时它会隐藏且不生效。 |
| 在预览区滚轮 / 拖动 / 双击 | 缩放、平移、还原缩放。 |
| **删除已选** / **一键删除全部** | 清理相册（会二次确认）。 |

拍照结果保存在脚本同级目录：中文系统下是 `拍照结果/`，其他语言下是 `Captures/`，文件名为 `试卷_<时间戳>.jpg` / `scan_<时间戳>.jpg`。

### 调翻页检测的灵敏度

**没有对应的命令行参数**，这些检测参数就是 `flipscan.py` 顶部的几个常量，改完重启即可：

| 常量 | 默认值 | 含义 |
|---|---|---|
| `MOTION_THRESHOLD` | `300` | 超过这个运动量才算「画面在动」。 |
| `MIN_FLIP_MOTION` | `8000` | 一次动作的峰值运动量达到多少才判定为翻页。 |
| `MIN_FLIP_SECONDS` | `0.3` | 短于这个时长的动作不算翻页。 |
| `STABLE_SECONDS` | `0.8` | 画面要稳定多久才按快门。 |
| `DIFF_THRESH` | `35` | 逐像素差分阈值。 |
| `MOTION_SMOOTH` | `6` | 运动信号的平滑帧数。 |
| `DETECT_RES` | `(480, 360)` | 检测用的分辨率（预览和成片仍是原始分辨率）。 |

误触发多就把 `MIN_FLIP_MOTION` 调大，翻了页却不拍就调小；拍到的图还在晃就把 `STABLE_SECONDS` 调大。

### AI 分析（可选）

在 `analyze_papers.py` 顶部的 `API_KEY` 填入 Gemini Key（在 <https://aistudio.google.com> → *Get API key* 获取）。**填了真实 Key 之后不要把这个文件提交进 git。**

它用的是 `gemini-2.5-flash`，内置的中文提示词让模型给出试卷的题目列表、涉及知识点、重点难点和常见易错点；想分析别的内容改同文件里的 `PROMPT` 即可。`--images` 支持 `.jpg`、`.png`、`.pdf`；`--video` 会先上传文件并等 Gemini 处理完再分析。

分析是独立的一步，拍照窗口不会自动调用它。

### 自动升级

高拍仪可以就地更新，不需要重新 clone。

- **自动（默认开）。** 启动几秒后到 GitHub 查最新 Release。有新版且安装目录可写时，
  **后台静默下载并安装**（不发桌面通知）。照片与 `~/.config/flipscan/` 会保留；重启程序后
  才跑新文件。没网、超时、不可写等失败安静跳过。
- **关掉。** **设置 → 其他 → 自动更新**（默认开）；安装时也会问一次。关掉后只有手动检查才联网。
- **手动。** **设置 → 其他 → 现在检查更新**立刻查并可安装（git `pull --ff-only` 或源码包覆盖，
  替换前会在安装目录旁备份）。

如果装在 `/opt` 这类系统目录里，就地更新需要写权限，改用 `sudo` 重跑 `install.sh` 即可。

### 良田（Eloam）用户须知

如果厂商官方软件正占用摄像头，程序会自动暂停它、用完后自动恢复；其他品牌被占用时会弹窗询问是否接管。
